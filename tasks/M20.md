# M20 — Autonomous Planning & World State Management

## Objective
Build the autonomous planning subsystem that decomposes high-level user goals into executable execution plans, and the world state subsystem that maintains a live model of the device environment for context-aware task execution.

## Milestone Structure
M20 is split into two subsystems delivered as independent chunks, both within `modules/automation/src/`:

### M20 Subsystem 1 — Planner (COMPLETE ✅)
- `planner.rs` — `Goal`, `ExecutionStep`, `ExecutionPlan`, `Capability` (13 variants), `PlanValidation`, `Planner`
- Heuristic goal decomposition covering 14+ goal types: brightness, volume/mute, screenshot, click/tap, type/enter text, search/find, remember/note/save, open/launch/start app, lock device, wifi, bluetooth, DND, fallback to AI (`RunAI`)
- Dependency graph with Kahn's algorithm topological sort + cycle detection
- `ready_steps()` — find steps whose dependencies are satisfied
- Builder-configured `Planner`: `with_max_steps()`, `with_default_timeout()`, `with_default_retry()`
- Helper functions: `extract_number()`, `extract_quoted()`, `extract_after()`
- 23 unit tests covering all goal patterns, topological sort (empty/linear/parallel/cycle), validation (valid/missing-dep/duplicate-id/cycle), ready_steps, goal context, planner configuration

### M20 Subsystem 2 — World State (PENDING)
- World state model (device state, app state, screen state, network state)
- World state store with diff tracking and subscriptions
- Integration with Planner for context-aware goal decomposition
- Permissions and privacy filtering
